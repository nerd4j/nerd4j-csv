nerd4j-csv

Framework for read, write CSV files and perform mapping between custom Java data models and CSV records.
So far this is the quickest, easiest and most configurable open source Java framework for CSV file manipulation.

Motivation

We found us to face the need to parse and manipulate every day thousands of CSV files containing gigabytes of data. Each file was different in terms of number of records, number and type of columns, quoting chars, separator chars, escaping policies, etc. We tried out several Java framework, both open source and not, but no one has met our needs of speed and customization. So we decided to write our own framework pointing on maximizing performance and providing a wide configurability.

During the development we take advantage from the nice job done by the Super-CSV developers providing a Model-Binding layer that performs out of the box the validation and conversion of a CSV cell and the binding of the parsed value into the provided data model.

Features

The nerd4j-csv framework is divided into two levels:

  1. a simple and very fast low level API that provides parsing and formatting from and to CSV sources.
  2. an easy to use and high configurable high level API that provides binding from and to custom data models.

The framework also provides a quick and easy configuration system and the possibility to access the internal registry to inject custom made components.

Low level parsing and formatting

First of all the nerd4j-csv framework provides a very low level API for parsing and formatting CSV files. The low level parser is twice as fast as the quickest CSV framework we tried out and provides a useful method to skip the parsing of columns that are not needed. It is possible to configure line and field separators, quoting characters and any other special char. The low level API also provides a custom escaping system (not CSV standard).

Model binding

On top of the parser/formatter API there is a Reader/Writer API that performs data binding from and to a custom data model. A CSVWriter uses a CSVParser to read the CSV source and transforms each record in the CSV source into the required data model. The nerd4j-csv framework provides three builtin model binders: the CSVToArrayBinder, the CSVToMapBinder and the CSVToBeanBinder, the last one uses reflection to populate the fields of a given POJO. By implementing the CSVToModelBinder interface is possible to register custom binders if needed.

Validation and conversion

Readers and Writers perform validation and conversion during the data binding. The nerd4j-csv framework provides validators and converters for all the most common use cases, but it is also possible to implement the CSVFieldValidator and CSVFieldConverter interfaces to write custom handlers and use them in the framework

The Registry

All builtin and custom components are registered into a common registry. The registry accepts implementations of the CSVRegistryEntryProvider interface. Those are classes able to provide the right instance of a component when requested. It is possible to register any type of component by providing the related implementation of the CSVRegistryEntryProvider interface.

Configurability

The nerd4j-csv framework can be configured through both annotations and XML and also provides the possibility to inherit configurations. So you can perform a quick and easy configuration by annotating your bean and than override or extend such configuration in the XML file.
