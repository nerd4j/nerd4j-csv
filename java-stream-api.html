NERD4J-CSV

So far this is the quickest, easiest and most configurable open source Java framework for CSV manipulation.

Support for Java 8 stream API

CSV Reader

Since version 1.2.0 support has been added for the new Java 8 Stream API. Now it's possible to use the CSVReader as the target of the "for-each-loop" statement and even to get a Stream of CSVProcessOutcomes. All methods have been designed to encourage functional programming and Java 8 Lambdas.

For instance you can use the CSVReader as the target of the "for-each-loop" statement using lambdas to inject beaviour on read success or read error. 

for( CSVProcessOutcome<Model> read : csvReader )
{
    read.success( model -> /* do something on success */ )
        .error( error -> /* do something on error */ )
        .then( context -> /* do something regardless result */ );
}

Another more affective way to use the CSVReader is by using the java.util.stream.Stream support. For example you can write something like this: 

csvReader.stream().peek( read ->
    read.error( error -> /* log error */ )
)
.filter( CSVProcessOutcome::isSuccess )
.map( CSVProcessOutcome::getModel )
.forEach( model -> /* do something */ );

It's a very clean a readable way to read a CSV source and perform some work on it's content.
Note. There is only one drawback on this approach. Since the CSV source can be anything: a string, a file but also a remote service response, the CSVReader is a single-use object and so are also the related Iterator and Stream.

So the CSVReader implements Iterable but unlike other iterable objects cannot be traversed more than once. If done, it will result in an empty Iterator or Stream. This behavior is not expected and can lead to issues tricky to be found. To enphatize this misbehavior a CSVSingleUseViolationException will be thrown if csvReader.iterator() or csvReader.stream() are called more than once.

CSV Writer

Since version 1.2.0 support has been added for the new Java 8 Stream API. Now it's possible to use the CSVWriter in a functional way to drain elements from Iterators, Spliterators and Streams.

Unfortunately the new Java 8 Stream API does not provide a way to flush all the elements of a stream into an OutputStream. A Stream can be "collected" or "reduced" but not drained. To adapt one of those methods to be used with a CSVWriter is possible but not elegant and probably misleading.

Therefore we decided to reverse the logic and make the CSVWriter to get a Stream to be drained into a CSV destination.

If you have no special needs all you need to do is this: 

csvWriter.drain( stream ).intoCSV()

If you want to be shure that write operations was successful or you need to do some additional work on each write you can proceed like this: 

csvWriter.drain( stream )
.forEach( write ->
    write.success( model -> /* do something on success */ )
         .error( error -> /* do something on error */ )
         .then( context -> /* do something regardless result */ )
)
.intoCSV();
